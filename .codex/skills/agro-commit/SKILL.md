# Commit Workflow Для Agro-Manager

## Когда применять

- Пользователь просит закоммитить изменения.
- Нужно зафиксировать итог работ одним шагом (не тестировать и не рефакторить дополнительно).
- Требуется соблюсти понятный русскоязычный стиль сообщений коммита.

## Минимальный порядок

- Не делать ничего лишнего перед коммитом: только `git add`, затем `git commit`.
- Проставить сообщение в повелительном/кратком стиле на русском.
- Для единичного коммита писать одну осмысленную тему, без суффиксов и англицизмов.

## Рекомендуемая форма сообщения

- Формат: `Короткое описание выполненных изменений`
- Пример: `Обновлены зависимости и добавлен скрипт выгрузки снимков экрана`

## Проверочный минимум

- `git status --short` перед коммитом для контроля, что коммитируются нужные файлы.
- После коммита — `git status` для проверки чистоты рабочей директории.
