# Материалы для защиты диплома по проекту AgroScope

## Готовый комплект: слайды + текст доклада

Ниже готовый контур для 12 слайдов. Его можно скопировать в PowerPoint, Google Slides или любой генератор презентаций.

### Слайд 1 — Титул
**На слайде:**  
AgroScope — цифровой центр управления хозяйством  
допуск по дипломной работе, автор, наставник, дата

**Текст сказа:**  
«Добрый день. Меня зовут ..., в работе защищаю дипломный проект AgroScope — веб-приложение для планирования и контроля полевых работ на сельхозпредприятии. Защита построена как мини-система для агронома и руководителя: карта полей, календарь задач, отчёты, рисковые сигналы и генерация DOCX-документов.»

### Слайд 2 — Проблема, которую закрываем
**На слайде:**  
- Распыление данных о полях, операциях и урожае по разным таблицам  
- Нет единой точки управления сроками работ и ответственностью  
- Риск просрочки и потери прозрачности по документам

**Текст сказа:**  
«На практике у хозяйств часто есть отдельные списки полей, личные таблицы и заметки, из-за чего теряются сроки и сложно увидеть общую картину по сезону. AgroScope делает это в одном месте: здесь лежат поля, календарь работ и отчеты, а интерфейс показывает, что уже выполнено, а что просрочено.»

### Слайд 3 — Архитектура решения
**На слайде:**  
- Frontend: React + TypeScript + FSD  
- Backend: Node.js + Express API  
- Данные: in-memory JSON-массивы для MVP  
- REST-взаимодействие и отдельный backend на `http://localhost:3001`

**Текст сказа:**  
«Архитектура разделена на frontend и backend. Во фронте используется FSD-структура: app, shared, entities, widgets, pages, components, features. На сервере — Express со своими маршрутизаторами: поля, операции, дашборды, отчёты, рекомендации и документы. В текущей версии данные берутся из локальных массивов, что ускорило разработку MVP и демонстрацию функциональности.»

### Слайд 4 — Роли и сценарии входа
**На слайде:**  
- Гость: просмотр данных  
- Агроном: редактирование полей и задач  
- Менеджер (расширение роли в модели)
- Аутентификация в демо: `sessionStorage`

**Текст сказа:**  
«У нас реализована ролевая модель. Гость может быстро посмотреть состояние хозяйства, а агроном получает рабочие возможности — редактировать параметры поля и добавлять задачи. Это помогает показать разграничение доступа уже на уровне UI и ролей, которое затем легко масштабировать до серверных правил и реальной авторизации.»

### Слайд 5 — Карта и карточки полей
**На слайде:**  
- Экран `/map` с интерактивной картой на Leaflet  
- Цветовая индикация культур  
- Выбор поля -> подробная карточка + детализация операций

**Текст сказа:**  
«Ключевой блок — визуальный контроль территории. Карта показывает реальные полигоны полей, их привязку и текущую культуру цветом. Через карту и боковую панель можно быстро перейти к нужной карточке поля и увидеть исторические операции и урожайность.»

### Слайд 6 — Календарь работ
**На слайде:**  
- Планирование по датам и типам операций  
- Фильтры по типу задач  
- Перенос дат и смена статусов для агронома

**Текст сказа:**  
«Календарь используется как операционный центр: ячейки недель, список предстоящих работ и быстрые действия по изменению статуса. Для агронома доступно создание новой задачи и корректировка времени/даты выполнения — это снижает риск просрочки и делает план понятным на уровне недели.»

### Слайд 7 — Журнал операций и поиск
**На слайде:**  
- Табличный список всех операций  
- Фильтры по полю, типу, статусу, датам  
- Удобная сортировка и обратная связь по состоянию

**Текст сказа:**  
«Если менеджеру нужна аудитная история в более формальном виде, используется журнал операций. Он покрывает весь срок с поиском и фильтрацией, поэтому можно быстро получить срез, например только просроченные, только по одному полю или только запланированные обработки.»

### Слайд 8 — Отчеты
**На слайде:**  
- Отчет по хозяйству и по конкретному полю  
- Статистика по типам и статусам операций  
- Динамика урожайности и выводы на текущий период

**Текст сказа:**  
«Модуль отчётности позволяет оперативно получать сводку по работе: сколько операций, по каким полям, какие статусы и каков баланс по урожайности. Это удобный блок для управленческого обсуждения и для передачи в бухгалтерию или планирование следующего этапа сезона.»

### Слайд 9 — Риски и погода
**На слайде:**  
- Автоматический центр рекомендаций  
- Сортировка рекомендаций по критичности  
- Блок погоды на 7 дней по центроиду полей

**Текст сказа:**  
«Чтобы не только планировать «что» и «когда», но и предупреждать, добавлен центр рисков. Он показывает просрочки и задачи с высоким сроковым риском, а также прогнозную погоду, важную при выборе окна для обработки, посева или внесения удобрений.»

### Слайд 10 — Документы
**На слайде:**  
- Загрузка DOCX шаблонов  
- Валидация токенов шаблона  
- Генерация документов по операциям/периодам

**Текст сказа:**  
«DOCX-модуль делает проект особенно практичным для реальной хозяйственной практики. Можно загрузить шаблон, проверить, какие токены распознаны, и сгенерировать документ в выбранном режиме. Для MVP это закрывает поток подготовки отчётов и актов без ручного переоформления данных.»

### Слайд 11 — Что уже сделано и что в MVP
**На слайде:**  
- Базовая эксплуатация: логин, карта, календарь, отчеты, документы  
- Единый API и общая модель данных  
- Порог MVP: ограниченная безопасность и хранение в памяти

**Текст сказа:**  
«В рамках диплома реализован полноценный рабочий минимум. Есть продуманный путь пользователя, единый backend-контур и UI с ролью пользователей. Для MVP принято решение о демонстрационной авторизации и in-memory данных, чтобы показать архитектуру и сценарии без инфраструктурных сложностей запуска.»

### Слайд 12 — Что дальше (реальный выпуск)
**На слайде:**  
- PostgreSQL + миграции данных  
- JWT + серверные права для всех операций  
- История изменений и аудит  
- Нотификации и мобильная адаптация рабочих экранов

**Текст сказа:**  
«Планы масштабирования стандартные: добавить БД и реальную авторизацию, внедрить аудит и журнал изменений, подключить push/Email напоминания, расширить мобильную работу и отчётность. Это позволит проекту выйти за рамки демо-версии и стать рабочим инструментом хозяйства.»

### Слайд 13 — Заключение и демонстрация
**На слайде:**  
- Комплексное решение для ежедневного управления  
- Единый контур для решений и отчётности  
- Готов к доработке под реальные процессы хозяйства

**Текст сказа:**  
«В сумме AgroScope закрывает цепочку: от планирования и мониторинга до отчётности и документов. Это ускоряет координацию и делает состояние хозяйства прозрачным на уровне дня. Готов пройтись по демонстрации экранов и показать ключевые сценарии в живом приложении.»

## Время выступления
Рекомендуемое время: 9–12 минут + 2–3 минуты вопросов.
